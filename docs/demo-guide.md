# 演示流程

这份文档用于面试、作品集录屏或给别人快速体验项目。

## 准备工作

确认 Docker Desktop 已启动，然后在项目根目录双击：

```text
start-formal.bat
```

正式版访问地址：

```text
http://127.0.0.1:3000
```

样例 CSV：

```text
samples/sample_leads.csv
```

## 标准演示路径

### 1. 仪表盘

打开首页，先说明这个项目不是单一工具，而是 AI 自动化控制台。

重点展示：

- 今日任务数
- 成功数
- 部分成功数
- 失败数
- 异常模块
- 最近任务

可以这样讲：

> 这个页面是运行状态入口，平台会把每次自动化流程的结果沉淀下来，方便判断今天跑了多少任务、哪些失败、哪些模块异常。

### 2. 功能管理

进入“功能管理”。

重点展示：

- 模块列表
- 模块启用/停用
- 模块健康状态
- 模块提供的能力，例如 `table.write`、`lead.normalize`
- 消息通知模块提供 `message.send`

可以这样讲：

> 我没有把业务流程写死到页面里，而是把功能拆成模块。比如飞书同步模块提供的是 `table.write` 能力，将来可以替换成 CRM 或 Notion。

### 3. 配置中心

进入“配置中心”，选择“飞书同步”。

重点展示配置项：

- `appId`
- `appSecret`
- `appToken`
- `leadTableId`
- `customerTableId`

可以这样讲：

> 模块的配置由后台统一管理，密钥不会写死在代码里。飞书配置缺失时，工作流不会崩溃，而是降级为本地保存。

### 4. CSV 上传

进入“CSV 上传”，选择：

```text
samples/sample_leads.csv
```

点击运行。

预期结果：

- 线索写入本地 `leads`
- 客户归并写入本地 `customers`
- 如果飞书配置正确，会同步到飞书线索明细表和客户表
- 如果飞书不可用，工作流显示 `partial_success`

### 5. 上传历史

进入“上传历史”。

重点展示：

- 文件名
- 运行 ID
- 处理行数
- 线索数量
- 客户数量
- 同步表
- 新增/更新数量
- 错误原因

可以这样讲：

> 这个页面解决的是追溯问题。我可以知道历史上传过哪些表，每次处理了多少数据，同步到哪些外部表，失败原因是什么。

### 6. 任务日志

进入“任务日志”。

重点展示每一步：

- `file.upload`
- `lead.normalize`
- `customer.merge`
- `table.write`
- `message.send`

可以这样讲：

> 每个能力调用都会记录日志，包括模块、能力、输入摘要、输出摘要、耗时、状态和错误信息。这样自动化流程出问题时，不需要靠猜。

### 7. 数据中心

进入“数据中心”。

重点展示：

- 一条客户咨询是一条线索
- 同一客户会归并到客户表
- 客户可以有多条线索
- 具体问题不会被合并丢掉

可以这样讲：

> 处理人员可以先看客户表判断客户整体情况，再进入线索明细表处理具体问题。

## 降级演示

如果要展示系统稳定性，可以这样做：

1. 打开“功能管理”。
2. 停用“飞书同步”模块。
3. 再上传一次 CSV。
4. 打开“工作流运行”或“上传历史”。
5. 查看状态为 `partial_success`。
6. 打开“数据中心”，确认线索和客户仍然写入本地。

讲解重点：

> 外部模块失败不影响核心数据处理。飞书停用时，同步能力不可用，但本地导入、线索清洗、客户归并仍然可用。

## 飞书机器人通知演示

可以用飞书群自定义机器人展示第二个插件能力：

1. 在飞书群里添加“自定义机器人”。
2. 复制机器人 webhook，格式通常类似 `https://open.feishu.cn/open-apis/bot/v2/hook/...`。
3. 如果机器人开启安全设置，建议先用“自定义关键词”，关键词填 `AI自动化平台`。
4. 打开“功能管理”。
5. 启用“消息通知”模块。
6. 打开“配置中心”，选择“消息通知”。
7. 把飞书机器人 webhook 填到 `webhookUrl`。
8. 停用“飞书同步”模块。
9. 上传一次 CSV。
10. 打开“任务日志”，查看 `message.send` 记录。

预期结果：

- 工作流状态为 `partial_success`
- 本地线索和客户仍然写入成功
- `table.write` 记录为跳过或失败
- `message.send` 记录通知发送成功、失败或跳过
- 飞书群里收到一条以“AI自动化平台通知”开头的文本消息

讲解重点：

> 飞书同步和消息通知是两个独立模块。飞书失败不会影响本地处理，通知失败也不会反过来影响主工作流。

## 重复上传演示

同一个 CSV 可以重复上传。

预期结果：

- 本地线索按规则去重
- 客户表重新归并
- 飞书已同步过的记录会更新，不会重复新增

讲解重点：

> 系统通过 `external_record_mappings` 保存本地记录和飞书 record_id 的关系，所以再次同步时可以更新外部记录。
